# Rollbacks Directory

This directory contains the rollback tracking and history for the TinyCloud GitOps platform.

## Files

| File | Purpose |
|------|---------|
| `rollbacks.yaml` | Structured audit trail of all rollbacks and restores per app |

## Structure

```yaml
apps:
  <app-name>:
    currentStatus: normal | rollback
    activeRollback: <id> | null
    history:
      - id: rb-<app>-<timestamp>
        type: rollback
        timestamp: "ISO-8601"
        targetRevision: <git-sha>
        targetImage: <full-image-with-sha>
        previousRevision: <git-sha>
        previousImage: <full-image-with-sha>
        reason: <string>
        rollbackBranch: rollback/<app>
        initiatedBy: <user>
      - id: rs-<app>-<timestamp>
        type: restore
        timestamp: "ISO-8601"
        restoredToRevision: <git-sha>
        restoredToImage: <full-image-with-sha>
        reason: <string>
        initiatedBy: <user>
```

## How It's Updated

The `scripts/rollback.sh` and `scripts/restore.sh` scripts automatically update this file and commit the changes to `main`.

**Do not edit this file manually.** Use the scripts or the future API instead.

## Audit Trail

Every rollback and restore creates:
1. A Git commit to `main` with structured metadata in the commit message
2. An entry in this YAML file
3. A rollback branch (`rollback/<app>`) at the target revision

This gives you **three independent audit trails**:
- Git log: `git log --grep="rollback(" --oneline`
- This YAML file: human and machine readable
- Argo CD UI: deployment history
